## 1. バグ修正：ハイライト保存「スレッド情報なし」エラー

### 症状
スレッドモーダル内で「Analyze Local Context (LLM)」を実行し、分析テキストを選択後「ハイライトとして保存」ボタンを押すと `❌ エラー：スレッド情報なし` が表示され、保存できない。

### 根本原因
`/api/conversations/{id}` エンドポイントのSQLで、`messages` テーブルへの SELECT に `conversation_id` カラムが含まれていなかった。その結果、フロントエンドの `currentThreadMessages` 配列に `conversation_id` キーが存在せず、`anchorMsg.conversation_id` が `undefined` → 文字列 `"undefined"` としてセットされ、バリデーションで弾かれていた。

### 修正ファイル

| ファイル | 変更内容 |
|---|---|
| `src/app.py` | `/api/conversations/{id}` SQLに `conversation_id` カラムを追加 |
| `src/static/js/main.js` | `triggerLocalAnalysis` に `resolvedConvId` フォールバックを追加 |
| `src/static/js/main.js` | `openThreadModal` で `_openedConvId` をキャッシュ |

## 2. 機能追加：Semantic Intelligence セクションの高度化

### 背景
Semantic Intelligence (LLM Analysis) の各出力（Overall Context / Dichotomy & Relations / New Definitions）について以下の課題があった。
- 出力根拠が不明（どのメッセージから生成されたか）
- テキストのコピー・ダウンロード不可
- 分析テキスト内の概念語からドリルダウン検索ができない

### 追加機能

#### ① 根拠の可視化（Evidence Source）
- 分析結果末尾に「📚 根拠（Evidence Source）」セクションを追加
- LLM分析に使用した元メッセージを会話タイトル・日付・本文冒頭で一覧表示
- 各スニペットをクリックするとスレッドモーダルが起動し、発言元の文脈へ遷移

#### ② コピー機能（Copy per Section）
- 各セクション（Overall Context / Dichotomy & Relations / New Definitions）にマウスオーバーで📋ボタンが出現
- クリックで当該セクションのテキストをクリップボードへコピー（2秒後に自動リセット）
- 「📋 全体コピー」ボタンで3セクションをMarkdown形式でまとめてコピー

#### ③ キーワードドリルダウン（Semantic Drill-down）
- LLM分析テキスト内の5文字以上の語句をクリック可能に（ホバーで点線アンダーライン表示）
- クリックするとその語句で再検索を発火（performSearch）

### 変更ファイル

| ファイル | 変更内容 |
|---|---|
| `src/app.py` | `/api/semantic_context` SQLに `m.id`、`m.conversation_id` を追加 |
| `src/static/js/main.js` | `fetchSemanticContext` レンダリングを全面改修 |
| `src/static/js/main.js` | グローバルイベント委譲に `.llm-keyword`・`.sem-evidence-item` ハンドラを追加 |
| `src/static/css/style.css` | `.sem-section`・`.sem-copy-btn`・`.sem-evidence-item`・`.llm-keyword` スタイルを追加 |

## 4. 技術的討議・将来仕様の確定

### 意味空間ビジュアライザー（フェーズ2）
bbycroft / thinkmachine 型の意味空間ビジュアライザーをフェーズ2の核心機能として位置付けた。既存の `vectors.npz`（bge-m3埋め込み）を活用し、UMAPによる次元削減・共起重みのAttention的可視化をD3.js / Three.jsで実装する方針を確定。

### UI改修の設計思想
現行の文字列ベースのドリルダウンが System 2（高認知負荷）に依存していることを認識。フェーズ2では二重過程理論を適用した5原則（空間的エンコーディング・前注意的特徴・プログレッシブ・ディスクロージャー・明確なアフォーダンス・文脈保持）に基づくUIを設計する。
