# 要件定義 (Requirements)

## 1. 目的と概要
ChatGPTの過去ログエクスポートから得た巨大なJSON形式のデータに対し、高速な全文検索およびNLPを用いた傾向分析・トピック抽出機能を備えたアプリケーションを構築する。目的は過去の思考の探索と再利用、及び知見の発見を促進すること。

## 2. システム要件

### 機能要件 (Functional Requirements)
- **ログデータの取り込み**: 複数の `conversations-xxx.json` をパースし、SQLite3 にインデクシングする (`build_db.py`)。
- **全文検索 (Full-Text Search)**: ユーザー入力キーワードをもとに、過去のメッセージ内から検索し、ヒットした会話IDやメッセージ内容を表示する。
- **ハイライト＆NLPタグ機能 (Glasp的機能)**: チャット内の興味のある文章を選択してマークアップし、保存できる。保存時にNLPで分類名称・タグを自動付与する。
- **形態素解析とインデックス化**: 検索の精度向上のため、また表示用データ抽出のために、メッセージ内容を `Janome` などで分かち書きし、名詞等のキーワードを保存する。
- **NLPによる分析**: 特定の検索結果や特定の期間について頻出単語群を集計し、ワードクラウド的もしくはグラフ的にUIに返す。
- **UIでの可視化**: チャットの検索フォーム、結果のタイムライン表示バー、そして分析結果を描画するダッシュボードを備える。

### 非機能要件 (Non-Functional Requirements)
- スキーマ構造は検索パフォーマンスを最適化するため、SQLite FTS5を使用。ただし日本語分かち書きインデックスを事前に作成して検索する。
- UIレスポンスは素早く、快適に操作可能 (SPA構成)。
- **デザイン**: 非常にモダンでプレミアムな、グラスモーフィズム(Glassmorphism)やダークモードを採用。

## 3. インターフェース要件
- 検索入力フィールド（クエリ、日付などでのフィルタ可能）
- リスト形式またはタイムライン形式での検索結果一覧
- ダッシュボード領域（頻出キーワードのチャート等）
