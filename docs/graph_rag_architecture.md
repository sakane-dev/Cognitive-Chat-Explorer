## GraphRAG 拡張アーキテクチャ構想

現在の「FTS（全文検索）＋形態素共起分析」に加え、**ローカルLLM（Ollama）**を活用して高度な意味ネットワーク（ナレッジグラフ）を自動構築する「GraphRAG」への拡張構想です。

### 1. 解決すべき課題
単純な「形式知」という単語の検索だけでは、「形式知と暗黙知がどのように対立しているか」「形式知がどのアーキテクチャで具体化（implements）されたか」というような**構造的な文脈（セマンティクス）**の検索や推論は困難でした。

### 2. GraphRAGのアプローチ
1. **テキストのセマンティックチャンキング**
   長いチャットログを意味のまとまり（チャンク）に分割します。
2. **エンティティと関係性（トリプル）の抽出**
   Ollamaで動作しているローカルLLMに対して、チャンクごとに「エンティティ(Entity)」とそれらの「関係性(Relationship)」を抽出させます。
   *例:* `[形式知] --(opposes)--> [暗黙知]`
3. **グラフデータベース（Knowledge Graph）の構築**
   これをNeo4jなどのグラフDB、あるいはSQLiteの専用スキーマ（`entities`, `relationships`テーブル）に保存します。
4. **ベクトル検索との融合**
   抽出されたエンティティの説明（description）をベクトル化してFAISS等に蓄積し、ユーザーの検索ワードに対する「意味的類似度」と「グラフ上の隣接ノード」の両方から回答を生成します。

### 3. 実験用パイロットプログラム (`src/extract_graph.py`)
まずはSQLiteに簡易的なグラフ構造を保存するPythonスクリプトを作成しました。
OllamaのJSONモードを利用し、既存のデータベースからテキストを読み込んで、エンティティとリレーションを抽出するプロトタイプです。

#### 使い方
```powershell
# Ollamaが動いている状態で実行
python src/extract_graph.py --limit 5 --model llama3 --api-url http://localhost:11434
```
これにより、新しいデータベース `knowledge_graph.db` が生成され、概念と概念の繋がり（セマンティック・ネットワーク）が可視化可能な形式で抽出・保存されます。
